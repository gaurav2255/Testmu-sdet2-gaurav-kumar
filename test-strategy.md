# Test Strategy

## Coverage Approach
- UI: Login, Dashboard, Form validation
- API: CRUD, error handling, response validation
- Integration: API + UI combined flow

---

## Why this approach
- Covers critical business flows
- Validates both frontend and backend layers
- Ensures real-world end-to-end scenarios

---

## What I would cover next
- Performance testing
- Security testing
- Accessibility testing
- Visual regression

---

## Top 3 Risks
1. Test environment instability
2. Flaky locators due to UI changes
3. Dependency on test data consistency
